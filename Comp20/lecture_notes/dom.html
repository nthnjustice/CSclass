Document Object Model (DOM)

  • The big idea: a structural representation a la tree of objects that represents the HTML document; a standard for how to get, change, add, or delete HTML elements.
  • A standard programming interface for HTML (API)
  • Allows you to view and modify page elements in script code after page has loaded. This is browser independent.
  • Very powerful: allows progressive enhancements of pages, and responsive interactions on the client-side
  • Standardized by the W3C
  • Each document is a hierarchy
  • Evolution:
    • DOM 1.0 - limited mutability
    • DOM 2.0 - full mutability
    • DOM 3.0+ - manipulation of child elements

Consider the Example

<!DOCTYPE html>
<html>
<head>
<title>Page Title</title>
<meta http-equiv="content-type" content="text/html; charset=utf-8" />
<meta http-equiv="Content-Language" content="en-us" />
</head>
<body>
<h1>This is a heading</h1>
<p>A paragraph with a <a href="http://www.google.com/">link</a>.</p>
<ul>
<li>a list item</li>
<li>another list item</li>
<li>a third list item</li>
</ul>
</body>
</html>

Nodes

  • Element nodes - Can have children and/or attributes; example: HTML tags. WARNING! Not all element nodes contains text!
  • Text nodes - A child within an element node; cannot have children; example: text in a block element
  • Attribute nodes - A child within an element node; cannot have children; example: attribute-value pair
