Two Versions of CSS

  • Version 2 (CSS2) - What you have seen so far; approved by the W3C
  • Version 3 (CSS3) - Visual enhancements (e.g., rounded corners via border-radius property), media queries, new selectors, new pseudo-classes; backwards compatible with CSS2; not officially a standard with the W3C

CSS2 Media Types

  • CSS2 supported only two different types of media: print and screen via media types
    • Screen (default): <link rel="stylesheet" type="text/css" href="style.css" media="screen" />
    • Printer: <link rel="stylesheet" type="text/css" href="printfriendly.css" media="print" />
  • Alas, no support for mobile screens!

Getting Around the Real Resolution Issue

  • Specify metadata for viewport
  • The crux of the matter: http://dev.opera.com/articles/view/an-introduction-to-meta-viewport-and-viewport/
  • Must be placed before linking stylesheets to page!
  • That is: <meta name="viewport" content="width=device-width; initial-scale=1.0; maximum-scale=1.0; user-scalable=0;" />
  • The above ensures that the user isn't able to zoom in or out the website

CSS3 Media Queries: A New Hope

  • Currently implemented in Safari, Firefox, and Opera
  • The idea: allow authors to customize a stylesheet based on some well-known properties, such as the size of the screen that is rendering the page
  • Example 1, stylesheet to be used by the iPhone's browser and not any PC browser: <link rel="stylesheet" href="handheld.css" media="only screen and (max-device-width:480px)"/>
  • Example 2, stylesheet to be used by everything else: <link media="print, screen and (min-device-width: 480px)" href="navigation1.css" rel="stylesheet" />
  • For the iPad / tablet, refer to article Using CSS Media Queries to Style Your iPhone and iPad HTML (O'Reilly Media)
  • Media queries syntax via Mozilla: https://developer.mozilla.org/en-US/docs/CSS/Media_queries
